Input Data Format

Inputs

  • Genomic data of the query sample: mutations alone, gene expression alone, and mutation plus gene expression. The input genomic data are analyzed for drug response prediction (Module 1) and similar samples (Module 2). 
  • Sample type of the query sample: cell line or tumor. Sample type determines which database is used to generate reference ranges for prediction results (cell line: Cancer Cell Line Encyclopedia [CCLE]; tumor: The Cancer Genome Atlas [TCGA]) and to search for similar samples.  
  • Data type (gene expression): FPKM or TPM.  
  • Log Transformed (gene expression): Whether the input data are already transformed to the log2(data+1) scale.

Genomics Data Format

shinyDeepDR accommodates common file formats for gene mutations and expression data: 

Mutations:

  • Mutation Annotation Format (MAF) file: We recommend following the maf format of cBioPortal or TCGA that contains three columns under the column name of "Hugo_Symbol", "Variant_Classification", and "Tumor_Sample_Barcode". Our tool internally filters and only uses four  types of nonsynonymous mutations, including "Missense_Mutation", "Nonsense_Mutation", "Frame_Shift_Ins", and "Frame_Shift_Del", which are used for DeepDR (Chiu et al. BMC Medical Genomics 2019).
  • Copy and paste the gene symbols carrying mutation in the text box: Please make sure to include only genes carrying the aforementioned four types of mutations: missense and nonsense mutations, and frameshift insertions and deletions, as they are the mutations that DeepDR was trained with.

    • Given either type of input, our tool extracts data of 18,281 genes that are used by DeepDR. Unmapped genes are imputed by the median value (e.g., 0 or 1) across the 622 cell lines used for constructing the DeepDR models.

Gene Expression:
  • txt file: The txt file should contain only two columns, with the first and second columns denoting gene symbols and expression levels (TPM or FPKM values; raw data scale or log2(data+1)), respectively. The txt file should contain a header. Since DeepDR is built with log2(TPM+1) values, we provide a conversion of FPKM values to TPM, as well as conversion to the log2(data+1) scale. For the quantification of gene expression, we strongly recommend to use the TCGA GDC pipeline  "mRNA Expression Workflow" with the "Original" setting that utilizes the GENCODE v22 annotation to minimize the number of unmapped genes DeepDR.

    • After the txt file is uploaded, our tool extracts data of 15,363 genes that are used by DeepDR. Unmapped genes are imputed by the median value across the 622 cell lines used for constructing the DeepDR models.
